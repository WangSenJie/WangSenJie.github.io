Vue 简介

Vue 是什么

Vue 是一个用来构建用户界面的 JavaScript 框架.

如果前面学的 HTML、CSS 和 JavaScript 是前端开发的基础三件套, 那么 Vue 可以理解成: 它是在这三件套之上, 提供了一套更高效的组织方式.

你可以先粗暴地把它理解成:

  • HTML 用来写页面结构
  • CSS 用来写页面样式
  • JavaScript 用来写交互逻辑
  • Vue 用来把这些内容更清楚地组织起来

所以 Vue 不是替代前端三件套, 而是建立在它们之上的一套开发方式.

为什么要用 Vue

当页面非常简单时, 只用原生 HTML、CSS 和 JavaScript 就够了.

但当页面越来越复杂时, 代码往往会出现这些问题:

  • 页面结构越来越大
  • 交互逻辑越来越多
  • 手动操作 DOM 很繁琐
  • 同样的页面块容易重复写

这时候 Vue 的价值就出来了.

它主要帮你解决这些问题:

  • 用组件拆分页面
  • 用数据驱动页面更新
  • 更方便地处理事件和表单
  • 让代码更容易维护

你可以把它理解成:
页面复杂之后, Vue 能让前端代码没那么乱.

什么是组件

组件是 Vue 里非常重要的概念.

你可以把组件理解成: 页面中的一个独立小模块.

比如一个页面可以拆成:

  • 顶部导航栏
  • 侧边栏
  • 文章卡片
  • 评论区
  • 登录框

这些都可以做成组件.

组件的好处是:

  • 每一块功能更独立
  • 可以重复使用
  • 页面结构更清楚
  • 后续修改时影响范围更小

例如一个按钮组件、一个卡片组件, 可以在多个页面里重复使用, 不需要每次都从头写一遍.

所以 Vue 的一个核心思路就是:
把大页面拆成很多小组件.

什么是数据驱动视图

这是 Vue 最核心的思想之一.

所谓“数据驱动视图”, 可以简单理解成:

  • 数据变了
  • 页面会自动跟着变

在原生 JavaScript 里, 你可能需要手动:

  • 选中元素
  • 修改 textContent
  • 处理页面更新

小结

  • Vue 是一个构建用户界面的 JavaScript 框架
  • 它建立在 HTML、CSS 和 JavaScript 之上
  • 它擅长用组件拆页面
  • 它的核心思路是“数据变, 界面跟着变”
  • 最常见的模板语法是 {{ }}、: 和 @
  • 最常见的指令是 v-if、v-for 和 v-model
