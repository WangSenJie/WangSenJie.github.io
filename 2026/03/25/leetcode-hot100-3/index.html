寻找岛屿数量

思路

深度优先遍历 DFS. 当遇到 grid[i][j]='1' 的时候, 代表此处是陆地, 于是岛屿数量+1, 于此同时, DFS遍历此位置上下左右元素, 遇到 0 或者边界则结束, 否则将遇到的 1 变成 0, 防止干扰下一座陆地的寻找.

腐烂的橘子

在给定的 m x n 网格 grid 中，每个单元格可以有以下三个值之一：

  • 值 0 代表空单元格；
  • 值 1 代表新鲜橘子；
  • 值 2 代表腐烂的橘子。

每分钟，腐烂的橘子周围 4 个方向上相邻的新鲜橘子都会腐烂。

返回直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 -1。

思路

  • 统计新鲜的橘子数量
  • 存储腐烂的橘子的坐标位置 queue
  • 每一分钟, pop queue 中的腐烂橘子坐标, 分上下左右四个位置腐烂周围的橘子, 再更新 queue 中的腐烂橘子的坐标 (此时老一批腐烂橘子的坐标已经全部弹出), 随后进行下一分钟的腐烂过程

(207) 课程表

你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses - 1. 在选修某些课程之前需要一些先修课程. 先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，表示如果要学习课程 ai 则必须先学习课程 bi.

  • 例如，先修课程对 [0, 1] 表示: 想要学习课程 0, 你需要先完成课程 1.

请你判断是否可能完成所有课程的学习? 如果可以, 返回 true; 否则, 返回 false.

思路

即判断是否是有向无环图.

  • 构造课程的入度表, 如果某课程有 n 门先修课程, 则入度为 n; 如果没有先修课程, 则入度为 0.
  • 构造后续课程表(字典功能), 记录课程 i 的后续课程, 方便后续索引查找.
  • 将入度为 0 的课程记录到队列 queue 中, 它们视为最开始修的课程, 因为没有先修课程.
  • 模拟学习课程
    • 修了第 i 门课程, 则 pop 该课程, 同时课程数量减 1.
    • 查找第 i 门课程的后续课程, 将它们的入度 -1. 同时如果此时它们的入度为 0, 则将其 append 到 queue.
  • 判断最后的课程数量, 如果数量非 0, 说明有环, False.

(208) 实现 Trie (前缀树)

Trie（发音类似 “try”）或者说前缀树是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这一数据结构有相当多的应用情景，例如自动补全和拼写检查。

请你实现 Trie 类：

  • Trie() 初始化前缀树对象。
  • void insert(String word) 向前缀树中插入字符串 word 。
  • boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
  • boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为 prefix ，返回 true ；否则，返回 false 。

思路

必须清楚这是一个树结构

  • 初始化一个长度为 26 的列表 self.tree = [None]*26 (表示 26 个字母), 每个位置事实上都是一个 Trie() 结构. 且设定 self.is_end = False, 代表此时这个 26 个位置并不是任何单词的结尾.
  • 以 app 说明 insert 过程, 初始化 node = self, 代表现在这棵树是在 node 位置
    • 先传入字母 a, 对应 index=0 位置, 判断这个位置上是否为 None
      • 如果为 None, 则在这个位置上创建一个新的 Trie(), 执行 Node = node.tree[0];
      • 如果不是 None, 即这个位置之前已经有单词存过, 那么只需执行 Node = node.tree[0], 跳到下面的分支.
    • 接着传入字母 p, 同样按上述步骤执行.
    • 最后传入 p, 也同样执行.
    • 执行完最后一个 p 以后, 设定 node.is_end = True, 代表这个分支是单词的结尾.
  • 以 app 说明 search 过程
    • 先传入字母 a, 对应 index=0 位置, 判断这个位置上是否为 None
      • 如果为 None, 则直接返回 False, 说明这个树结构里一定没有 app 这个单词, 因为它甚至没有 a;
      • 如果不是 None, 即这个位置之前已经有单词存过, 那么只需执行 Node = node.tree[0], 跳到下面的分支.
    • 接着传入字母 p, 同样按上述步骤执行.
    • 最后传入 p, 也同样执行.
    • 执行完最后一个 p 以后, 说明树结构一定有 app, 此时只需判断 node.is_end 是否是 True, 即最后一个 p 是否是结尾.
  • 以 app 说明 startsWith 过程
    • 先传入字母 a, 对应 index=0 位置, 判断这个位置上是否为 None
      • 如果为 None, 则直接返回 False, 说明这个树结构里一定没有 app 这个单词, 因为它甚至没有 a;
      • 如果不是 None, 即这个位置之前已经有单词存过, 那么只需执行 Node = node.tree[0], 跳到下面的分支.
    • 接着传入字母 p, 同样按上述步骤执行.
    • 最后传入 p, 也同样执行.
    • 执行完最后一个 p 以后, 说明树结构一定有 app 开头的单词, 此时只需返回 True.
