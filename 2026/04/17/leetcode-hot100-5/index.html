(560) 和为 的子数组

给你一个整数数组 nums 和一个整数 k, 请你统计并返回该数组中和为 k 的子数组的个数.

子数组是数组中元素的连续非空序列.

利用前缀和和Hash表, 其中前缀和表示数组的起点到当前位置的元素和.
如果用 代表从起点到 位置的元素和, 则第 位置到第 位置的元素和为 .

  • 初始化当前前缀和 Prefix <- 0, 初始化满足条件的非空连续序列的数量 count <- 0, 定义一个 Hash 表 hashtable = {prefix: count} (键为记录的前缀和, 值为前缀和出现的次数).
  • 遍历 nums 中的所有元素 num.
    • 更新当前前缀和 Prefix <- Prefix + num, 要找是否存在和为 k 的连续序列, 只需 hashtable 中找是否存在一个键为 Prefix - k. 如果存在的话, 则说明我们能找到这样的连续非空序列, 且这样的连续序列数量为 hastable[Prefix - k], 更新 count <- count + hastable[Prefix - k]. 若不存在一个键为 Prefix - k, 则说明没找到这样的非空连续序列, 进入下一轮循环.
    • 每一轮循环结束前, 更新 hashtable[Prefix] += 1

(239) 滑动窗口最大值

给你一个整数数组 nums, 有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧. 你只可以看到在滑动窗口内的 k 个数字. 滑动窗口每次只向右移动一位.

返回滑动窗口中的最大值.

使用队列 queue, 队列中存放元素的下标, 且队列中的元素保持严格单调递减, 即 queue[0] 一定对应队列中的最大元素.

  • 遍历 nums 中的所有元素 num, 设其下标为 i
    • 从后往前遍历 queue 中的元素, 如果比 num 小, 则将其 pop, 最后得到的 queue 是保持递减.
    • 将 num 的下标 i 插入到 queue 中
    • 如果 i-queue[0] >= k, 说明 queue[0] 已经不在当前窗口中, 将其 pop
    • 如果 i >= k-1, 则说明窗口已经生成, 将 queue[0] 加入到 result 中

(76) 最小覆盖子串

给定两个字符串 s 和 t, 长度分别是 m 和 n, 返回 s 中的最短窗口子串, 使得该子串包含 t 中的每一个字符 (包括重复字符). 如果没有这样的子串, 返回空字符串 "" .

测试用例保证答案唯一.

采用滑动窗口和计数器.

  • 初始化窗口左边界 l <- 0, 初始化最小覆盖子串的左右边界 l* <- -1, r* <- len(s), 初始化计数器 count_t = Counter(t), count_s = Counter()
  • 将右边界从 r = 0 开始向右移动
    • 更新计数器 count_s[s[r]] += 1
    • 如果 count_s >= count_t, 则说明此时窗口中的子串可以覆盖 t, 下面判断该覆盖子串是否比以往的最小覆盖子串更小, 即 是否成立, 若成立, 则更新 l*, r* 的值. 随后将窗口左边界往右移动, 即 count_s[s[l]] -= 1, l += 1.
  • 如果执行完循环后, 仍有 l = -1, 则说明一定没有覆盖子串, 此时返回空字符串.
