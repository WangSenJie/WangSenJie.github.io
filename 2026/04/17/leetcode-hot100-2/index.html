LeetCode Hot100 —— 双指针

LeetCode-hot100 中“双指针”部分的题解

双指针

(283) 移动零

给定一个数组 nums, 编写一个函数将所有 0 移动到数组的末尾, 同时保持非零元素的相对顺序.

请注意, 必须在不复制数组的情况下原地对数组进行操作.

用一个 index_zero 指向零元素的 index, 初始化 index_zero = -1, 用另一个指针 i 遍历 nums 数组

  • 如果 i 位置是 0, 且 index_zero = -1, 则赋值 index_zero <- i
  • 如果 i 位置是 0, 且 index_zero != -1, 则说明 index_zero - i 之间都是零元素, 此时继续向右移动指针 i 去找非零元素
  • 如果 i 位置非 0, 且 index_zero = -1, 说明指针 i 还没遇到零元素, 因此只需继续向右移动指针 i
  • 如果 i 位置非 0, 且 index_zero != -1, 说明指针 index_zero - i-1 之间都是零元素, 此时交换 index_zero 和 i 位置的元素, 将零元素往后移, 随后更新 index_zero <- index_zero + 1 (此时 index_zero + 1 一定是零元素)

(11) 盛最多水的容器

给定一个长度为 n 的整数数组 height. 有 n 条垂线, 第 i 条线的两个端点是 (i, 0) 和 (i, height[i]).

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水.

返回容器可以储存的最大水量.

说明: 你不能倾斜容器.

用 双指针 思路, 其中 left 指针指向容器的左侧, right 指向容器的右侧.

  • 初始化 left = 0, right = n-1, V_max = 0
  • 在 left < right 的前提下执行循环
    • 每次循环计算当前容器的体积 V = min(height[left], height[right]) * (right-left), 更新 V_max = max(V_max, V)
    • 如果 height[left] < height[right], 则将 left 向右移动; 反之, 将 right 向左移动.

$\star$(15) 三数之和

给你一个整数数组 nums, 判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0. 请你返回所有和为 0 且不重复的三元组.

注意: 答案中不可以包含重复的三元组.

注意，输出的顺序和三元组的顺序并不重要。

采用 双指针 思想

  • 如果 len(nums) < 3, 则一定找不到满足条件的三数组合, 直接返回空
  • 对数组 nums 进行排序 (升序)
  • 遍历 nums 中的所有数 (i, num), i 为 num 的下标
    • 如果 num > 0, 则说明后续所有数都为正数, 那么一定找不到三数之和为 0 的组合, 直接推出循环;
    • 如果 num 和前一轮循环的数一样, 则跳过此轮循环
    • 如果 num <= 0, 则构造指针 left = i+1, right = len(nums) - 1
      • 如果 num + nums[left] + nums[right] < 0, 则需要增大组合数的和, 只需执行 left += 1; 反之, 则执行 right -= 1
      • 随后, 要注意更新 left 和 right. 将 left 指针不断往右移, 直至指向第一个不同的点; 将 right 指针不断往左移, 直至指向第一个不同的点

(42) 接雨水

给定 n 个非负整数表示每个宽度为 1 的柱子的高度图, 计算按此排列的柱子, 下雨之后能接多少雨水.

采用双指针 left 和 right, 其中 left 从最左边的柱子开始向右移动, right 从最右边的柱子开始向左移动

  • 初始化接的雨水量 result = 0, left_max = 0, right_max = 0, 其中 left_max 记录左边柱子的最大高度, right_max 记录右边柱子的最大高度
  • 在 left < right 的前提下进行循环
    • 每次更新 left_max = max(left_max, height[left]), right_max = max(right_max, height[right])
    • 如果 height[left] < height[right], 则记录 result <- result + (left_max-height[left]), 同时向右移动 left <- left + 1; 反之, 则记录 result <- result + (right_max-height[right]), 同时向右移动 right <- right - 1
